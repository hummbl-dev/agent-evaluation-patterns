# v0.1 Scope and Boundary

This document defines the scope, non-goals, adoption boundary, review expectations, non-canon posture, required packet pieces, and adjacent repo links for v0.1 of Agent Evaluation Patterns.

## Status

Candidate namespace. Exploratory unless later adopted through a reviewed governance path.

## Scope

v0.1 defines the minimum, reviewable artifacts needed to describe agent evaluation as version-controlled source material:

- A definition of what an evaluation packet is.
- A minimal JSON schema for an evaluation packet.
- One valid example and one valid fixture.
- One invalid fixture demonstrating a missing required field.
- A prior-art and adjacent-ecosystem reference document.
- A receipt describing what was produced and why.

v0.1 covers evaluation packets only. It does not cover execution runbooks, deployment, or operational telemetry.

## Non-goals

- This repo is not a universal standard.
- This repo does not claim ownership of the broader field or terminology.
- This repo does not canonize HUMMBL/BaseN/Ownward concepts unless explicitly marked and audited.
- This repo must not include private, internal, or secret operational content.
- v0.1 does not define adoption criteria, validation tooling, or governance process. Those are deferred to later versions.

## Adoption boundary

v0.1 artifacts are candidate material. Adoption is not implied by inclusion. Any later adoption must go through a reviewed governance path that does not yet exist in this repo.

## Review expectations

- Each artifact must be small and reviewable.
- Each artifact must preserve provenance and evidence.
- Each artifact must mark candidate status clearly.
- Schemas must be valid JSON Schema.
- Examples and fixtures must match the schema or fail it intentionally.

## Non-canon posture

Nothing in v0.1 is canon. Names, fields, and structure are proposals. Prior art is cited to avoid novelty and ownership overclaims. Adjacent ecosystem references are descriptive, not endorsements.

## Required packet pieces

A v0.1 evaluation packet must contain:

1. `schemaVersion` — the schema version this packet conforms to.
2. `packetStatus` — the lifecycle status of the packet (e.g. candidate, draft, adopted).
3. `evaluationManifest` — identity, title, version, and scope of the evaluation.
4. `authority` — who authored and who owns the packet.
5. `evaluationContract` — the task suite, rubric, judge configuration, and statistical requirements.
6. `receiptRequirements` — what must be recorded when the packet is executed.

## Adjacent repo links

This repo is one node in a small set of candidate repositories. Links are descriptive and not endorsements:

- `hummbl-dev/agent-evaluation-patterns` — this repo.
- `hummbl-dev/hummbl` — candidate HUMMBL concepts (separate governance).
- `hummbl-dev/basen` — candidate BaseN concepts (separate governance).
- `hummbl-dev/ownward` — candidate Ownward concepts (separate governance).

Each adjacent repo retains its own governance. Nothing here canonizes them.
